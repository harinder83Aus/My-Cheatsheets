üìÅ Repository Creation

Initialize a new repository

Clone an existing repository

Example: git clone https://github.com/user/repo.git

üìù Basic Workflow

Check repository status
git status
Add files to staging area
git add <file>
git add .
Commit changes
git commit -m "message"
Example: git commit -m "Add user authentication"

üåø Branching

List branches
git branch
Create new branch
git branch <branch-name>
Switch to branch
git checkout <branch-name>
git switch <branch-name>
Create and switch to new branch
git checkout -b <branch-name>
Example: git checkout -b feature/login

üîÑ Merging

Merge branch into current branch
git merge <branch-name>
Delete branch
git branch -d <branch-name>
Force delete branch
git branch -D <branch-name>
Example: git merge feature/login

üåê Remote Operations

Add remote repository
git remote add origin <url>
List remotes
git remote -v
Push to remote
git push origin <branch>
Pull from remote
git pull origin <branch>
Example: git push origin main

üìä History & Logs

View commit history
git log
git log --oneline
git log --graph
Show changes in commit
git show <commit-hash>
View file changes
git diff
git diff --staged

‚Ü©Ô∏è Undoing Changes

Unstage files
git reset <file>
Discard changes in working directory

Reset to previous commit
git reset --hard HEAD~1
Revert commit
git revert <commit-hash>
‚ö†Ô∏è Be careful with --hard reset, it permanently deletes changes!

üì¶ Stashing

Stash changes

git stash save "message"
List stashes

Apply stash
git stash apply
git stash pop
Drop stash
git stash drop

üè∑Ô∏è Tagging

Create tag
git tag <tag-name>
git tag -a <tag-name> -m "message"
List tags
git tag
Push tags
git push origin <tag-name>
git push origin --tags
Example: git tag -a v1.0 -m "Version 1.0 release"

‚ö° Useful Aliases

Set up helpful shortcuts

git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.unstage 'reset HEAD --'
üí° Now you can use 'git st' instead of 'git status'!

üîß Advanced Commands

Interactive rebase

Cherry pick commit

Find commit that introduced bug

Search in commit messages

Show who changed what

üí° Pro Tips:
  • Use git status frequently to check your repository state
  • Write meaningful commit messages that explain why you made changes
  • Use branches for feature development and keep main/master stable
  • Pull before pushing to avoid conflicts
  • Use .gitignore to exclude files you don't want to track
