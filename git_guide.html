Git Lifecycle Overview

Git has three main areas where your files exist:

  1. Working Directory - Your actual files on disk
  2. Staging Area (Index) - A preparation area for your next commit
  3. Repository - Where Git stores the committed history

Essential Git Commands by Category

Initial Setup

# Configure your identity (do this first!)
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# Initialize a new repository
git init

# Clone an existing repository
git clone <repository-url>

Basic Workflow Commands

# Check status of your files
git status

# Stage files for commit
git add <filename>          # Stage specific file
git add .                   # Stage all changes
git add *.js               # Stage all JavaScript files

# Commit changes
git commit -m "Your commit message"
git commit -am "Message"    # Stage and commit tracked files in one step

# View commit history
git log
git log --oneline          # Condensed view
git log --graph            # Visual branch representation

Branching and Merging

# List branches
git branch                 # Local branches
git branch -r             # Remote branches
git branch -a             # All branches

# Create and switch branches
git branch <branch-name>   # Create branch
git checkout <branch-name> # Switch to branch
git checkout -b <branch-name> # Create and switch in one command

# Modern alternative to checkout (Git 2.23+)
git switch <branch-name>
git switch -c <branch-name> # Create and switch

# Merge branches
git checkout main
git merge <branch-name>

# Delete branches
git branch -d <branch-name>    # Safe delete (merged only)
git branch -D <branch-name>    # Force delete

Remote Repository Commands

# Add remote repository
git remote add origin <repository-url>

# View remotes
git remote -v

# Push changes
git push origin <branch-name>
git push -u origin <branch-name>  # Set upstream tracking

# Pull changes
git pull origin <branch-name>
git pull                    # From tracked branch

# Fetch changes (without merging)
git fetch origin

Undoing Changes

# Unstage files
git reset <filename>
git reset                   # Unstage all

# Discard working directory changes
git checkout -- <filename>
git restore <filename>      # Modern alternative

# Undo last commit (keep changes in working directory)
git reset --soft HEAD~1

# Undo last commit (discard changes)
git reset --hard HEAD~1

# Revert a commit (creates new commit)
git revert <commit-hash>

Git Best Practices

Commit Messages

Follow this format:

Type: Brief description (50 characters max)

Longer explanation if needed (wrap at 72 characters)

- Use bullet points for multiple changes
- Reference issue numbers if applicable

Common types: feat, fix, docs, style, refactor, test, chore

Example:

feat: Add user authentication system

- Implement login/logout functionality
- Add password hashing with bcrypt
- Create user session management
- Add input validation for forms

Fixes #123

Branching Strategy

Feature Branch Workflow:

# Start new feature
git checkout main
git pull origin main
git checkout -b feature/user-authentication

# Work on feature, make commits
git add .
git commit -m "feat: Add login form"

# When done, merge back
git checkout main
git pull origin main        # Get latest changes
git merge feature/user-authentication
git push origin main
git branch -d feature/user-authentication

File Management

# Create .gitignore file to exclude files
echo "node_modules/" >> .gitignore
echo "*.log" >> .gitignore
echo ".env" >> .gitignore

# Remove file from tracking but keep locally
git rm --cached <filename>

Collaboration Best Practices

  • Always pull before pushing: 
    git pull origin main
git push origin main
  • Use descriptive branch names: 
    feature/payment-integration
bugfix/header-alignment
hotfix/security-vulnerability
  • Make atomic commits - each commit should represent one logical change
  • Review before committing: 
    git diff                    # See unstaged changes
git diff --staged          # See staged changes
git status                 # Overview of changes

Common Workflow Example

Here's a typical development session:

# 1. Start work session
git status
git pull origin main

# 2. Create feature branch
git checkout -b feature/add-search

# 3. Make changes to files
# ... edit files ...

# 4. Review and stage changes
git status
git diff
git add src/search.js
git add tests/search.test.js

# 5. Commit with good message
git commit -m "feat: Add search functionality

- Implement search algorithm
- Add search input component  
- Include basic search tests"

# 6. Push feature branch
git push -u origin feature/add-search

# 7. After code review, merge to main
git checkout main
git pull origin main
git merge feature/add-search
git push origin main

# 8. Clean up
git branch -d feature/add-search
git push origin --delete feature/add-search

Helpful Aliases

Add these to your Git config for efficiency:

git config --global alias.st status
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.unstage 'reset HEAD --'
git config --global alias.last 'log -1 HEAD'
git config --global alias.visual '!gitk'

Advanced Tips

Stash Work Temporarily

git stash                  # Save current work
git stash pop             # Restore stashed work
git stash list            # See all stashes

Interactive Staging

git add -p                # Stage chunks interactively

Amend Last Commit

git commit --amend        # Modify last commit message
git commit --amend --no-edit  # Add changes to last commit

Getting Started Advice

Start with the basic workflow and gradually incorporate more advanced features as you become comfortable. The key is consistent practice with good habits from the beginning.

Remember: Git is a powerful tool that becomes intuitive with practice. Don't be afraid to experiment in a test repository!

Final Notes
