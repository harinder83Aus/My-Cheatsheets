üìÑ Dockerfile
Basic Dockerfile Instructions
FROM image:tag
Set base image
WORKDIR /path/to/workdir
Set working directory
COPY source dest
Copy files from host to image
ADD source dest
Copy files (supports URLs and archives)
RUN command
Execute command during build
CMD ["executable", "param1", "param2"]
Default command to run
ENTRYPOINT ["executable", "param1", "param2"]
Configure container as executable
EXPOSE port
Inform Docker that container listens on port
ENV KEY=value
Set environment variable
VOLUME ["/path/to/volume"]
Create mount point
USER username
Set user for subsequent instructions
LABEL key=value
Add metadata to image
üéØ Example: Complete Dockerfile
# Use official node image as base
FROM node:16-alpine

# Set working directory
WORKDIR /app

# Copy package files
COPY package*.json ./

# Install dependencies
RUN npm ci --only=production

# Copy application code
COPY . .

# Create non-root user
RUN addgroup -g 1001 -S nodejs
RUN adduser -S nextjs -u 1001
USER nextjs

# Expose port
EXPOSE 3000

# Health check
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
CMD curl -f http://localhost:3000/health || exit 1

# Start application
CMD ["npm", "start"]
üí° Dockerfile Best Practices
‚Ä¢ Use multi-stage builds for smaller images
‚Ä¢ Leverage build cache by ordering instructions properly
‚Ä¢ Use .dockerignore to exclude unnecessary files
‚Ä¢ Run as non-root user for security
‚Ä¢ Use COPY instead of ADD unless you need URL/archive features
